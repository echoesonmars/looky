# Looky — платформа: PWA, mobile-first, карта функций

Документ связывает [functions.md](./functions.md) с экранами приложения, навигацией (dock) и требованиями к PWA. **Приоритет: мобильный телефон как основной сценарий;** на планшетах и десктопе — те же функции, но раскладка расширяется (двухколоночные макеты, боковая навигация вместо или вместе с dock).

---

## 1. Принципы

| Принцип | Реализация |
|--------|------------|
| **Mobile first** | Вёрстка и UX от 320px вверх; touch targets ≥ 44px; жесты (свайпы) на ключевых экранах. |
| **PWA** | Установка на домашний экран, `standalone`, офлайн-оболочка (shell), push там, где нужны алерты цен. |
| **Адаптивность** | На `md+` — опциональный сайдбар или второй столбец; dock может стать **левой навигацией** или **остаться снизу** (продуктовое решение). |
| **Без «стека» в UI** | Пользователь видит продуктовые названия экранов, не названия библиотек. |

---

## 2. Навигация: dock (мобильный) и большие экраны

### 2.1 Нижний dock (основной на телефоне)

Пять слотов (иконка + короткая подпись). Активный таб подсвечивается.

| # | Маршрут | Название для пользователя | Назначение |
|---|---------|---------------------------|------------|
| 1 | `/` или `/home` | **Главная** | Сводка: быстрые действия (примерка, стилист), избранное, дайджест. |
| 2 | `/discover` | **Лента** | Свайпы «как тиндер», визуальный поиск, рекомендации. |
| 3 | `/wardrobe` | **Гардероб** | Вещи пользователя, CPW, теги, эко-скоринг по вещи. |
| 4 | `/stylist` | **Стилист** | Чат с ИИ, капсула по бюджету (вход из чата или отдельный подэкран). |
| 5 | `/profile` | **Профиль** | Аккаунт, уведомления о ценах, настройки, PWA / установка. |

**Центральная кнопка «+»** (FAB над dock или внутри пятого таба): добавить вещь (камера / галерея) → поток с авто-тегами.

На **`md` (768px)+**: dock можно **зафиксировать снизу** или **перенести в левый сайдбар** с теми же пунктами + раскрытием подменю.

### 2.2 Вторичная навигация (без отдельного таба в dock)

- **Виртуальная примерка** — с главной и из карточки товара: `/try-on` (полноэкранный или модалка).
- **Генератор принтов / кастом** — `/create` или `/shop/custom` (высокая сложность, позже).
- **Авторизация** — `/login`, `/register` (модалки или полноэкранные страницы вне dock).

## 3. Карта страниц и функций из [functions.md](./functions.md)

### 🟢 Низкая сложность

| Функция | Где в приложении | Примечание |
|--------|-------------------|------------|
| **Трекер износа (CPW)** | `Гардероб` → карточка вещи → блок «Стоимость носки» + лог надеваний | CRUD событий «надел»; формула цена / число носков. |
| **Умный трекер цен (алерты)** | `Профиль` → «Цены» / «Уведомления» + список отслеживаемых товаров | Cron/API на бэкенде; **Web Push** в PWA при срабатывании. |
| **Tinder для одежды** | `Лента` — основной экран свайпов | Коллаборативная фильтрация; датасет лайков/дизлайков. |

### 🟡 Средняя сложность

| Функция | Где в приложении | Примечание |
|--------|-------------------|------------|
| **Auto-tagging при загрузке** | Поток **«Добавить вещь»** после выбора фото | Сегментация/классификация; сохранение тегов в карточке вещи. |
| **Эко-скоринг («умный состав»)** | `Гардероб` → карточка вещи → вкладка «Состав» | NLP по тексту состава; подсказки по износу/катышкам. |
| **Сбор капсулы по бюджету** | `Стилист` → сценарий «Подобрать образ до N ₽» или отдельный шаг в чате | Оптимизация набора + цены из парсеров/API. |
| **AI-стилист (чат)** | `Стилист` — основной UI | RAG по гардеробу + контекст (погода, событие). |

### 🔴 Высокая сложность

| Функция | Где в приложении | Примечание |
|--------|-------------------|------------|
| **Генератор принтов** | `Создать` / раздел магазина кастома | Text-to-image + print-on-demand. |
| **Визуальный поиск** | `Лента` — кнопка камеры / «Найти похожее» | Эмбеддинги + векторный поиск по каталогам. |
| **Virtual Try-On** | `Главная` CTA + карточка товара → `/try-on` | Тяжёлый ML-пайплайн; на мобильном — прогресс и превью. |

## 4. Структура маршрутов (черновик)

```
/                     → маркетинговый лендинг (текущий) или редирект в /home для авторизованных
/home                 → главная приложения (с dock)
/discover             → свайпы + вход в визуальный поиск
/wardrobe             → список вещей
/wardrobe/[id]        → карточка: CPW, состав, теги
/stylist              → чат стилиста (+ капсула по бюджету)
/profile              → профиль, алерты цен, настройки
/try-on               → виртуальная примерка
/create               → кастом / принты (позже)
/login, /register     → без dock
```

API и воркеры остаются на бэкенде; фронт только потребляет эндпоинты.

## 5. PWA: что нужно реализовать

1. **`manifest.json`** / `manifest.webmanifest`: `name`, `short_name`, `icons` (192/512), `display: "standalone"`, `theme_color`, `background_color`, `start_url: "/home"` (или целевой экран после логина).
2. **Иконки** для iOS/Android (maskable).
3. **Service Worker** (Workbox или встроенный в Next): кэш статики и shell; стратегия для API — network-first или только онлайн для тяжёлых фич.
4. **Viewport / safe-area**: `env(safe-area-inset-*)` для dock и FAB под «чёлкой» и home indicator.
5. **Push** (опционально, для алертов цен): подписка из `Профиль`; VAPID на сервере.
6. **Установка**: баннер или пункт «Установить приложение» в профиле (`beforeinstallprompt`).

## 6. Адаптивность: поведение по ширине

| Диапазон | Навигация | Контент |
|----------|-----------|---------|
| **< 768px** | Нижний dock, полноширинные экраны | Одна колонка; свайпы на всю ширину. |
| **768px – 1024px** | Dock снизу или слева (узкий сайдбар) | Список + деталь в split-view где уместно (гардероб). |
| **≥ 1024px** | Сайдбар + при необходимости сохранение dock как дубль | Две-три колонки (лента + превью; гардероб + деталь чата). |

## 7. Статус реализации (обновлено)

**Сделано:** группа маршрутов `app/(app)` с layout: нижний dock (мобильный), левый сайдбар (`md+`), FAB «+», отступы под safe-area; страницы `/home`, `/discover` (встроен `SwipeSection`), `/wardrobe`, `/wardrobe/[id]`, `/wardrobe/add`, `/stylist`, `/profile`, `/try-on`; ссылка «приложение» с лендинга на `/home`; базовый PWA — `app/manifest.ts`, иконки SVG в `public/icons/`, `appleWebApp` и `themeColor` в корневом layout.

**Запланировано:** service worker и push; привязка гардероба к API и данным; чат стилиста и тяжёлые фичи (примерка, визуальный поиск).
